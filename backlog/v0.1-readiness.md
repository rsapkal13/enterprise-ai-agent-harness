# v0.1.0-alpha Readiness Checklist

This checklist tracks whether Enterprise AI Agent Harness is ready to tag `v0.1.0-alpha`.

`v0.1.0-alpha` is a specification-readiness milestone. It is not a production runtime release.

## Release Checks

- [x] README updated
- [x] Docs aligned
- [x] Roadmap aligned
- [x] Schemas compile
- [x] Example manifests validate
- [ ] CI passes
- [ ] Website links verified
- [x] Public/private boundary documented
- [x] No runtime overclaims
- [x] No private data
- [x] GitHub Issues created
- [ ] `v0.1.0-alpha` tag ready

## Notes

- CI should pass on the final commit before tagging.
- Website links should point to public repository pages and avoid claiming runtime features that are not part of v0.1.
- The release should contain reusable open-source concepts only: no real customer data, private APIs, credentials, internal architecture, or organisation-specific approval thresholds.
- The tag should be created only after the working tree is clean and validation has passed.

## Suggested Tag Commands

```bash
npm run validate
git status
git tag -a v0.1.0-alpha -m "v0.1.0-alpha spec release"
git push origin v0.1.0-alpha
```
