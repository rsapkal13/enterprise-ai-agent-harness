# v0.2 Readiness Checklist

Target: `v0.2 - Local Runtime Preview`

Status: **Ready to tag — pending green CI run on main**.

The v0.2 release should prove that the specification can be exercised locally through validation,
in-memory runtime components, mock tools, workflow state, policy decisions, trace/audit evidence,
and fictional examples. It does not need production infrastructure.

## Release Gates

- [x] v0.1 schemas and examples validate locally.
- [x] Local schema and example validation scripts exist.
- [x] Cross-reference validation exists for example manifests.
- [x] In-memory registry prototype exists.
- [x] YAML manifest loader exists.
- [x] Mock tool gateway exists.
- [x] Basic policy decision contract exists.
- [x] In-memory workflow engine exists.
- [x] Trace emitter exists.
- [x] In-memory audit sink exists.
- [x] Banking card-dispute example exists as fictional v0.2 journey.
- [x] Admin console prototype exists for registry, approval, evidence, and relationship graph review.
- [x] Public backlog has expanded v0.3+ coverage.
- [ ] Latest GitHub Actions `Validate` workflow passes on `main`.
- [x] `npm test` reliably executes package tests in CI and locally.
- [x] v0.2 scope is reflected in `README.md`, `ROADMAP.md`, and package READMEs.
- [ ] Open GitHub issues are closed or relabeled so v0.1/v0.2 status is accurate.
- [ ] `v0.2.0` tag is created after green CI.

## Known Blockers

### CI run on main

The GitHub Actions `Validate` workflow must pass on `main` before tagging. All trailing-whitespace
failures from earlier runs have been resolved. The `npm test` glob pattern (`packages/*/tests/*.test.js`)
is in place and all 167 tests pass locally across 53 suites. A clean CI run is the only remaining gate.

### GitHub issue triage

Issues #6–#35, #48, #49, #51, #52, #54, #55 are ready to close. Issues #25 and #50 stay open
(consent/approval model and link checker are scoped to v0.3+).
