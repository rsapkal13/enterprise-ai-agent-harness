# Design Principles

## Govern Capabilities, Not Prompts

Prompts are useful implementation details. The harness treats business capabilities, policies, tools, workflows, and audit records as first-class objects.

## Vendor Neutrality

The model should work with many model providers, agent frameworks, identity systems, workflow tools, and integration platforms.

## Explicit Risk

Every executable capability should declare risk, ownership, approval needs, and evaluation expectations.

## Auditable Execution

Every meaningful decision and system action should produce traceable evidence.

## Progressive Runtime

The specification should be useful before the runtime is complete. Registries and schemas come first.
