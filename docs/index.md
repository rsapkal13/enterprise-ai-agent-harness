# Documentation

Enterprise AI Agent Harness documents the control layer for governed AI execution.

Start with the vision, then read the architecture overview, then explore each concept page. The schemas and examples show how the model becomes portable configuration.

## Primary Sections

- Vision: why the harness exists
- Architecture: system boundaries and design principles
- Concepts: registry, policy, workflow, system, context, audit, UI, observability, and evaluation
- Guides: early authoring workflows for specification objects
- Governance: responsible AI, risk, audit, approval, and privacy models
- Schemas: human-readable schema notes linked to JSON Schema files
- Reference: lifecycle and versioning conventions for registry objects
